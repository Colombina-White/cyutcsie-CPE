# 1.cpp - 迴文排列 (Palindrome Permutation)

核心邏輯：將字串切為前半段，利用 next_permutation 找尋下一個排列，再鏡像生成後半段。

# 2.cpp - 質因數分解 (Prime Factorization)
試除法：從最小的質數 2 開始嘗試整除目標數 n。

範圍優化：只需要檢查到 sqrt{n} 即可。如果到sqrt{n}都無法整除，剩下的 n 必定為一個質數。

輸出順序：迭代版：使用 stringstream 將新找到的質因數拼接到字串前端，實現由大到小的輸出。

遞迴版：先進行遞迴呼叫再執行 cout，利用遞迴堆疊（Stack）的特性，達成倒序輸出的效果。 

sqrt{n}就是n開根號

大數處理：使用 long long 避免處理大於 2^31-1 的整數時發生溢位。

遞迴終止：當 n 變為 1 時必須停止遞迴，否則會造成無窮迴圈。

# 3.cpp - AL 字串刪除 (String Manipulation)
定位關鍵字：

標準庫版：使用 find_last_of("Aa") 找最後一個 A，再用 find_first_of("Ll", a_pos) 從 A 的位置向後找 L。

手寫版：外層 for 迴圈從陣列尾端掃描（n-1 到 0）來找 A，內層 while 則從 A 的索引往後掃描找 L。

字串刪除：

標準庫版：直接呼叫 s.erase()。

手寫版：將 L 之後的所有字元（包含結束符號 \0）往前搬移到 A 的位置，直接覆蓋掉要刪除的區段。

字元陣列結尾：手寫搬移字元後，務必在正確位置加上 \0，否則輸出的字串尾端會出現舊資料或亂碼。

索引同步：在刪除一段字元後，手寫版的 i = i（雖然在程式碼中看似無用）是用於提醒開發者檢查是否需要重新掃描目前位置。

# 4.cpp - 簡易計算機 (Arithmetic Expression)
循序運算：程式不考慮運算優先順序（先乘除後加減），而是由左至右看到一個運算子就執行一次計算。

浮點數與取整：使用 double 進行運算以保留精確度，最後輸出時加上 0.5 並轉型為 int 以達成四捨五入。

注意：除法檢查：在進行 / 運算前，必須檢查 next != 0 以避免程式崩潰。

結束條件：當讀取到 = 字元時，代表運算式結束，應跳出迴圈。

# 5.cpp - 雜亂文字中的數字加總 (Digit Extraction)
狀態切換：利用 has_num 布林值記錄目前是否正在「讀取數字中」。

數值組裝：當讀到字元 c 為數字時，透過 num = num * 10 + (c - '0') 將字元轉換為數值並累進位數。

atoll (標準庫)：將 C 字串 (char*) 轉換為 long long。

f.get(c) (手寫版)：逐一讀取檔案中的「每一個字元」（包含空白），確保不會跳過任何非數字字元。

# 6.cpp - 最大子陣列和 (Kadane's Algorithm)
Kadane 演算法：維護一個 curSum（包含當前數字的目前最大和）與 maxSum（全域最大和）。

決策點：如果 val 大於 curSum + val，代表先前的累計和是負擔，應從當前數字 val 重新開始計算。

# 7.cpp - 最小公倍數 (LCM)
兩兩運算：先算出前兩個數的 LCM，再將結果與第三個數算 LCM，依此類推。數學公式：LCM(a, b) = (a * b) / GCD(a, b)。

溢位控制：為了防止 a * b 超出 long long 範圍，實作時應先除再乘：(a / gcd) * b。

# 8.cpp - 字元行程編碼 (RLE Encoding)
相鄰比較：遍歷字串，若 s[i] == s[i+1] 則計數加 1，否則輸出計數與字元。

十六進位切換：使用 cout << hex;，之後所有輸出的整數都會自動轉為 16 進位格式。

# 9.cpp - 數值內插法 (Linear Interpolation)

# 11.cpp - ()[]{}所組成之字串

遇左括號：將該左括號推入堆疊（push）。

遇右括號：檢查堆疊是否為空（如果空了代表右括號多出來，錯誤）。

檢查堆疊頂端是否為相對應的左括號（如果對不上，錯誤）。

如果匹配成功，將堆疊頂端的左括號彈出（pop）。

結束檢查：所有字元掃描完後，堆疊必須是空的（如果堆疊還有剩，代表左括號多出來，錯誤）。

影響擴散：遇到 x 或 X 時，啟動內層迴圈將範圍內（前後 1 或 2 格）的 safe[j] 設為 0。

# 12.cpp - 中國餘數定理

計算總乘積 N： N = p1 x p2 x ..... x pn。

計算各別項的權重 Mi： Mi = N / pi（即除了自己以外的所有除數乘積）。

尋找模反元素 $y_i$： 尋找一個數字 $y_i$，滿足 $(M_i \times y_i) \equiv 1 \pmod{p_i}$。

構造最終解 $X$： $X = \sum (r_i \times M_i \times y_i)$。

取最小正整數解： $Ans = X \pmod N$。

# 14.cpp - 異常值

平均值 ($\mu$)：所有數值總和除以數量 $n$。

變異數 ($\sigma^2$)：每個數值與平均值之差的平方和，再除以 $n$。

$$\sigma^2 = \frac{\sum (x_i - \mu)^2}{n}$$

標準差 ($\sigma$)：變異數的算術平方根。

判定條件：

第一段判定： $|x_i - \mu| > \sigma$

第二段判定： $(x_i - \mu)^2 > \sigma^2$

數學上這兩者是等價的。

# 15.cpp - 身份證號碼

字母轉換

每個英文字母對應一個 2 位數的代碼（例如 A=10, B=11...，但 O=35, W=32 等不按順序）。

程式中的 getLetterVal 函式透過一個預設陣列 vals[] 精確地處理了這個映射關係。

字母加權

假設字母轉換後的數字為 $N$，其加權公式為：$$加權值 = (N \text{ 的十位數}) \times 1 + (N \text{ 的個位數}) \times 9$$

對應程式碼：sum = (n / 10) + (n % 10) * 9;

數字加權累加

剩餘的數字（前 8 碼）分別乘上遞減的權重（8, 7, 6, 5, 4, 3, 2, 1）:

第 1 位數字 $\times 8$

第 2 位數字 $\times 7$

...以此類推

對應程式碼：sum += (id[i] - '0') * (9 - i);（當 $i=1$ 時，權重為 $9-1=8$）

計算檢查碼

將上述所有加權和相加得到 sum，檢查碼為：

$$檢查碼 = (10 - (sum \pmod{10})) \pmod{10}$$

# 16.cpp - 數字拆解與計數

遍歷範圍

使用一個 for 迴圈，從 $0$ 一直檢查到 $n$。

注意：題目給出的 $n$ 最大可達 $10^8$（一億）。雖然暴力法在 $n=10^8$ 時執行時間會接近 1 秒左右，但對於一般的簡單評測來說通常是可以接受的。

數字拆解（各位數求和）

利用 while 迴圈進行數字拆解：

while (temp > 0) {

    sum += temp % 10; // 取得最後一位數（餘數）並累加
    
    temp /= 10;       // 將數字右移一位（去尾）
    
}

例如：若 temp = 123：

123 % 10 = 3，sum = 3，temp = 12

12 % 10 = 2，sum = 3 + 2 = 5，temp = 1

1 % 10 = 1，sum = 5 + 1 = 6，temp = 0（結束）

條件判斷與計數如果計算出來的 sum 等於目標值 k，則將計數器 count 加 1。

# 17.cpp - 存款複利率 

$P$：每年年初存入的本金。

$n$：存款年數。

$r$：年利率。

計算方式：
第一年結束：$Total = (0 + P) \times (1 + r)$

第二年結束：$Total = (第一年總額 + P) \times (1 + r)$

依此類推。

演算法邏輯:二分搜尋

設定區間：假設利率在 $0$ ($0\%$) 到 $1$ ($100\%$) 之間。

取中點 mid：測試當前的利率 mid。

模擬計算：用 mid 算出 $n$ 年後會得到多少錢。

調整區間：

如果算出來的錢 < 目標金額：代表利率太低了，將下限 low 移至 mid。

如果算出來的錢 > 目標金額：代表利率太高了，將上限 high 移至 mid。

重複執行：程式執行了 100 次循環，這能讓精確度達到極高（$2^{-100}$），遠超一般的浮點數需求。

# 20.cpp - 部分序列字串指原字串中刪掉一個以上字元且不改變原順序的部份字串。
## 1. 核心觀念：什麼是子序列？

子序列的定義是：從原字串中刪除零個或多個字元，且 不改變剩餘字元的相對順序 所產生的字串。

例如：abc 是 axbycz 的子序列。

例如：acb 不是 axbycz 的子序列（因為順序錯了）。

## 2. 程式碼邏輯詳解
程式碼使用 雙指標（Two Pointers） 的概念來比對兩個字串：

指標初始化

指標 i 指向字串 s1（我們要尋找的目標序列）。

迴圈指標 j 走訪字串 s2（原始較長的字串）。

del 字串用來記錄沒被選中的字元。

貪婪比對

程式遍歷 s2 的每個字元：

如果匹配：當 s1[i] == s2[j] 時，代表在 s2 中找到了 s1 所需的下一個字元，於是將 i 往後移動一位（i++）。

如果不匹配：代表 s2[j] 這個字元不是我們目前需要的，或是順序不對，於是將這個字元加入 del 字串中。

結果判定

當遍歷完 s2 後：

如果 i 的數值等於 s1 的長度，表示 s1 中的每一個字元都依序在 s2 中找到了對應，輸出 Yes 以及被刪除的字元 del。

否則，輸出 No。
# 21.cpp - 輸出最小的子數列長度
## 1. 數列生成邏輯
程式第一部分是在建立數列 $x$。它使用的不是隨機數，而是一種三階遞迴（類似費氏數列，但由前三項相加）：

前三項：直接設定 $x[1]=1, x[2]=2, x[3]=3$。

後續項：$x[i] = (x[i-3] + x[i-2] + x[i-1]) \pmod m + 1。

% m：確保數字落在 $0$ 到 $m-1$ 之間。

+ 1：平移區間，讓數字最終落在 $1$ 到 $m$ 之間。

這種產生方式在競賽中很常見，用來模擬一組看似亂序但可重複生成的測試資料。
## 2. 核心搜尋演算法：計數桶 
程式的第二部分是在找「包含 $1 \sim k$ 的最短前綴長度」。這裡運用了 計數桶 ：
cnt[1000] 陣列：記錄數字 $1$ 到 $k$ 出現的次數。

found 變數：這是一個「進度條」。我們不關心數字出現幾次，只關心「這類數字是否出現過」。

當我們讀到 $x[i]$，且 $x[i] \le k$ 時。

關鍵條件：if (cnt[x[i]] == 0)。如果這個數字是第一次出現，我們才讓 found++。

終止條件：一旦 found == k，代表集合 $\{1, 2, 3, \dots, k\}$ 裡的每個數字都至少出現過一次了。
